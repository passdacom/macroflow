# drag-drop-sequencer.md — 비주얼 플로우차트 시퀀서 스펙

---

## 1. 개념 정의

시퀀서는 단순한 "순서대로 실행" 큐가 아니다.
**매크로 JSON 파일들을 노드로 하는 비주얼 플로우차트 편집기 + 실행기**다.

```
[단순 순차 실행]           [조건 분기 플로우]
  login.json                  login.json
      ↓                           ↓
  query.json           color_check(로딩 완료?)
      ↓                    ↓            ↓
  export.json           query.json    오류처리.json
                             ↓
                        counter(100회?)
                         ↓        ↓
                     query.json  완료
```

두 가지 모두 같은 시퀀서 UI에서 편집한다.

---

## 2. 노드 타입

| 노드 타입 | 아이콘 | 설명 | 연결 출력 |
|---|---|---|---|
| macro | 📄 | 매크로 JSON 파일 실행 | success / failure 2개 |
| color_check | 🎨 | 픽셀 색 감지 대기 | match / timeout 2개 |
| counter | 🔢 | 반복 카운터 | continue / max_reached 2개 |
| wait_fixed | ⏱ | 고정 대기 | next 1개 |
| end | ⬛ | 플로우 종료 (success/error) | 없음 |

## 4. 노드 편집 인터랙션

### 노드 추가
- 팔레트에서 캔버스로 드래그앤드롭
- macro 노드: 드롭 시 파일 선택 다이얼로그 → JSON 파일 선택
- color_check 노드: 드롭 후 속성 패널에서 좌표·색상 설정

### 노드 연결
- 노드 우측 출력 포트(작은 원)에서 드래그 → 다음 노드 입력 포트로 연결
- 연결선 위에 조건 레이블 표시 (success / failure / match / timeout)
- 연결선 클릭 → 삭제 가능

### 노드 선택 및 편집
- 노드 클릭 → 하단 속성 패널에 설정값 표시
- 속성 패널에서 직접 수정 (파일 경로, 색상값, timeout 등)
- 노드 더블클릭 → macro 노드의 경우 내부 이벤트 에디터 열기

### 파일에서 불러오기
- 기존 JSON 파일을 캔버스로 드래그앤드롭 → macro 노드로 자동 추가
- 여러 파일 동시 드롭 → 세로로 자동 배치, success 연결 자동 생성 (단순 순차)

## 5. color_check 노드 설정 UI

속성 패널에서:

```
┌─────────────────────────────────┐
│  🎨 색 감지 설정                 │
│                                 │
│  감지 좌표                       │
│  X: [25.0]%   Y: [75.0]%       │
│  [화면에서 클릭해서 선택]         │
│                                 │
│  목표 색상                       │
│  [████] #FFFFFF  [현재 색 읽기]  │
│                                 │
│  허용 오차: [10] (±RGB)          │
│  확인 주기: [50] ms              │
│  최대 대기: [10000] ms           │
│                                 │
│  timeout 시: ○팝업 ●다음노드이동 │
└─────────────────────────────────┘
```

**"화면에서 클릭해서 선택" 기능:**
버튼 클릭 → 앱이 최소화 → 마우스로 화면 클릭 → 좌표 자동 입력 + 해당 픽셀 색 자동 읽기

**"현재 색 읽기" 기능:**
입력된 좌표의 현재 픽셀 색을 GetPixel로 즉시 읽어 목표 색상에 자동 입력.

## 7. 플로우 저장 및 불러오기

- 저장 형식: `.macroflow` (JSON, scripting-engine.md 포맷 참조)
- 저장 위치: 매크로 JSON 파일과 동일 폴더 권장
- 매크로 파일 경로: `.macroflow` 기준 상대 경로로 저장
  ```json
  "macro_path": "./login.json"   ← 절대 경로 금지
  ```
  → 폴더째 이동해도 경로 깨지지 않음

## 8. 단순 순차 실행 (기존 기능 유지)

플로우차트 편집기를 쓰기 싫은 경우를 위한 단순 모드:

```
파일 목록 영역에 JSON 파일들을 순서대로 드래그앤드롭
→ 위에서 아래로 순서대로 실행
→ 내부적으로 success 연결만 있는 단순 플로우차트로 처리
```

단순 모드와 플로우차트 모드는 동일한 `.macroflow` 포맷을 공유한다.
단순 모드에서 만든 파일을 플로우차트 편집기에서 열면 선형 그래프로 표시된다.
