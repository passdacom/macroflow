## 2. 버그 수정 완료

### BUG-01: 시퀀서 저장 — 다중 매크로 불러오기 실패 ✅
**현상**: 시퀀서에 2개 이상 매크로를 등록하고 `.macroflow` 파일로 저장 후 불러오면 첫 번째 매크로만 로드됨.  
**확정 원인**: 저장 시 매크로 사이에 간격이 있으면 `.macroflow`가 `MacroNode → WaitFixedNode → MacroNode` 형태가 된다. 기존 로드 루프는 `MacroNode`가 아닌 노드를 만나면 중단해서 첫 번째 매크로만 복원했다.
**해결 방식**:
- `script_engine.py`에 `iter_linear_macro_paths()` 추가
- `MacroNode`는 경로 수집 후 `next_on_success`로 이동
- `WaitFixedNode`는 `next`로 이동해 다음 매크로까지 계속 순회
- `EndNode` 또는 비선형 노드는 안전하게 순회 종료
- `sequencer.py`의 `_load_flow_from_path()`가 이 헬퍼를 사용하도록 변경
**수정 파일**: `src/macroflow/script_engine.py`, `src/macroflow/ui/sequencer.py`
**검증 방법**: `tests/test_sequencer_backlog.py::test_linear_macro_paths_walks_through_wait_nodes`

---

### BUG-02: 시퀀서 저장 버튼 미분리 ✅
**현상**: 시퀀서/플로우 탭의 저장 버튼이 항상 "다른 이름으로 저장" 동작만 수행. 에디터처럼 "저장(덮어쓰기)"와 "다른 이름으로 저장"이 분리되어야 함.  
**해결 방식**:
- 저장(Ctrl+S): 현재 `.macroflow` 파일에 덮어쓰기 확인 다이얼로그 후 저장
- 다른 이름으로 저장: 별도 버튼, 항상 파일 선택 다이얼로그
- 파일 미지정 상태에서 저장 클릭 시 → 다른 이름으로 저장으로 위임  
- 메인 창의 Ctrl+S/다른 이름 저장도 시퀀서 탭에서는 시퀀서 플로우 저장으로 라우팅
**수정 파일**: `src/macroflow/ui/sequencer.py`, `src/macroflow/ui/main_window.py`
**검증 방법**:
- `tests/test_sequencer_backlog.py::test_sequencer_exposes_save_and_save_as_actions`
- `tests/test_sequencer_backlog.py::test_main_save_shortcuts_route_to_sequencer_on_sequencer_tab`

## 3. 기능 개선 완료

### FEAT-01: 즐겨찾기 탭 활성화 시 녹화·재생 비활성화 ✅
**요구사항**: 즐겨찾기 탭이 선택된 상태에서는 F6(녹화 시작/중지), F7(재생) 단축키 및 툴바 버튼 비활성화.  
**이유**: 즐겨찾기 탭에서는 녹화/재생 대상 매크로가 없으므로, 실수로 누르는 것을 방지.  
**해결 방식**:
- `nativeEvent()`의 F7 처리에서 즐겨찾기 탭이면 재생하지 않고 반환
- `_toggle_playback()`에서도 즐겨찾기 탭이면 즉시 반환
- `_update_toolbar()`에서 즐겨찾기 탭의 재생 버튼을 비활성화
**수정 파일**: `src/macroflow/ui/main_window.py`
**검증 방법**: `tests/test_sequencer_backlog.py::test_favorites_tab_disables_play_toolbar_action`

---

### FEAT-02: 즐겨찾기 → 시퀀서 추가 시 탭 자동 전환 제거 ✅
**요구사항**: 즐겨찾기 우클릭 → "📋 시퀀서에 추가" 클릭 시 시퀀서 탭으로 자동 전환하지 않음.  
**이유**: 여러 매크로를 연속으로 시퀀서에 추가할 때 탭이 계속 전환되어 불편.  
**확정 원인**: 탭 전환은 `favorites.py`가 아니라 `main_window.py`의 `_add_favorite_to_sequencer()`에서 수행되고 있었다.
**해결 방식**: 시퀀서 추가 후 `self._tabs.setCurrentWidget(self._sequencer)` 호출 제거, 상태바 메시지만 유지
**수정 파일**: `src/macroflow/ui/main_window.py`
**검증 방법**: `tests/test_sequencer_backlog.py::test_favorite_add_to_sequencer_does_not_switch_tabs`

## 5. v1.2.0 구현 완료

### NEW-01: 텍스트 입력 추가 시 딜레이 설정 ✅
**현황**: `_insert_text_input` 다이얼로그가 텍스트만 입력받고, 딜레이는 1000ms로 고정 hardcode.  
**요구사항**: 다이얼로그에 "딜레이(ms)" 스핀박스 추가. 기본값 1000ms, 범위 0~30000ms.  
설정한 딜레이 값이 `delay_override_ms`로 저장되어, 에디터 딜레이 열에 표시되고 재생 시 반영.  
**수정 파일**: `editor.py` (`_insert_text_input`)

### NEW-02: 에디터 시간 열 — 절대/상대 전환 체크박스 ✅
**현황**: 에디터 "시간(ms)" 열은 항상 녹화 시작 기준 절대 시간 표시 (예: 100ms, 200ms, 350ms).  
**요구사항**: 툴바에 "⏱ 상대 시간" 체크박스 추가.

- **체크 OFF (기본)**: 기존 절대 시간 표시 유지
- **체크 ON**: 이전 이벤트 행과의 시간 차(델타) 표시
  - 첫 번째 행: 절대값 그대로 (이전 이벤트 없음)
  - 이후 각 행: `현재_이벤트.timestamp_ns - 이전_이벤트.timestamp_ns` (ms 변환)
  - 단, `delay_override_ms`가 설정된 경우 그 값 우선 표시 (딜레이 설정이 곧 간격)

예시: 1번 클릭 100ms, 2번 클릭 200ms → 상대 표시: 1번 100ms, 2번 100ms  
딜레이 설정 예시: delay_override_ms=500이면 상대 표시도 500ms  
**수정 파일**: `editor.py` (`_DisplayRow`, `_build_rows`, 테이블 렌더링, 툴바)

### NEW-03: 에디터 우클릭 → 클릭 이벤트 추가 ✅
**현황**: 우클릭 메뉴에 텍스트 입력 추가만 있음.  
**요구사항**: "🖱 클릭 추가(&L)..." 메뉴 항목 추가. 클릭 시 다이얼로그:

1. **위치**: X%, Y% 스핀박스 + "📍 화면에서 직접 지정 (F6으로 지정)" 버튼
   - F6 캡처 시 위치 자동 입력 (기존 `_edit_position`과 동일한 F6 캡처 메커니즘)
   - F6 캡처 시 해당 위치의 픽셀 색도 자동 저장 (`recorded_color` 설정)
2. **버튼 종류**: 라디오 버튼 — ○ 좌클릭 ○ 우클릭 ○ 더블클릭
3. **딜레이(ms)**: 스핀박스, 기본 1000ms
